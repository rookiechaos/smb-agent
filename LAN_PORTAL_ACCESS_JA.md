# LAN Portal Access Guide

この資料は、社内の Mac mini 上で動く SMB Agent Portal を、
経営者のパソコンからどのように開くかを説明するための
短い導入・引き渡し用ガイドです。

## 前提

標準構成は以下です。

- SMB Agent 本体は会社専用の Mac mini に導入
- Portal もその Mac mini 上で提供
- 経営者は自分の PC からブラウザで確認
- 基本は同一 Wi-Fi または同一社内 LAN 内で利用

つまり、通常は経営者の PC に専用アプリをインストールする必要はありません。

必要なのは:

- 社内ネットワーク接続
- ブラウザ
- 共有されたアクセス先アドレス

## 経営者が使う画面

経営者が主に使うのは以下です。

- `owner.html`

必要に応じて、最初の入口として:

- `index.html`

を案内しても構いません。

## 典型的な開き方

導入時に保守担当から、以下のような社内アドレスを共有します。

例:

- `http://192.168.1.20:PORT/portal/index.html`
- `http://192.168.1.20:PORT/portal/owner.html`

実際の IP アドレスとポート番号は、設置先の Mac mini に合わせて案内します。

## 経営者の操作手順

1. 社内 Wi-Fi または社内 LAN に接続する
2. Edge / Chrome などのブラウザを開く
3. 案内された URL を入力する
4. `owner` 画面をブックマークする
5. 必要時に再読込して最新状態を確認する

## 何が見えるか

経営者画面では、主に以下を確認します。

- ワークフローが稼働中か
- 承認待ちがあるか
- 完了したか
- 要確認のものがあるか
- API 利用量が月次予算の何％か

これは経営者向けの確認画面であり、
技術ログや内部の思考情報を見るための画面ではありません。

## トラブル時の確認

ページが開けない場合は、まず以下を確認します。

1. 経営者の PC が社内ネットワークに接続されているか
2. Mac mini の電源が入っているか
3. Portal 提供サービスが停止していないか
4. URL が正しいか

## よくある現象

### 1. ページが開かない

考えられる原因:

- 社内 LAN に入っていない
- Mac mini 側のサービス停止
- アドレス変更

対応:

- まず社内ネットワーク接続を確認
- 改善しない場合は保守担当へ連絡

### 2. 表示は開くが内容が古い

考えられる原因:

- ブラウザの表示が更新されていない

対応:

- 再読込する
- それでも変わらない場合は保守担当へ連絡

### 3. 要確認が出ている

意味:

- 何らかのワークフローで人の確認が必要

対応:

- 承認対象を確認する
- 必要なら保守担当または担当社員に連絡する

## 推奨運用

経営者向けには、以下の運用がわかりやすいです。

- `owner` 画面だけをブックマーク
- 日次または必要時に確認
- 承認待ちと要確認だけを重点的に見る

これにより、技術的な詳細を見なくても、
「今ちゃんと動いているか」を把握できます。

## 導入時に渡すべき内容

引き渡し時には、最低限以下を共有します。

- Portal URL
- ブックマーク対象ページ
- 問題時の連絡先
- 稼働確認の見方
- 承認待ちが出た時の対応方針

## 標準メッセージ例

経営者向けの引き渡し時には、次のように案内できます。

「このシステムは会社専用の Mac mini 上で動いています。  
経営者の方は PC にアプリを入れる必要はなく、社内ネットワーク上で
ブラウザから Portal を開くだけで、現在の進行状況や承認待ちを確認できます。」
